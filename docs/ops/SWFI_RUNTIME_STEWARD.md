# SWFI Runtime Steward

SWFI Runtime Steward is the always-on operating layer for the SWFI intelligence surface.

It does not:
- invent facts
- bypass trust or review gates
- publish unsupported AI output
- replace deterministic exports or audit logs

It does:
- observe official-source watches, connector health, export readiness, and review queues
- prioritize high-value research targets and "gold from mush" candidates
- route narrow-expert AI work through OpenAI first, then Anthropic, then Google Gemini
- enforce refusal outside SWFI's sovereign and institutional-capital scope
- write outcomes back into governed nuggets, profile signals, audit logs, and briefings

## Core Loop

1. Observe
- official-source watchlist
- connector probes
- MSCI export readiness
- profile repair queue
- governed nugget queue

2. Decide
- which sources should be checked first
- which findings are publishable now
- which findings require review
- which provider should answer inside the narrow-expert AI lane

3. Delegate
- source-grounded AI response generation
- nugget extraction and ranking
- review and promotion workflow
- export/report generation

4. Write Back
- nugget review ledger
- export audit log
- profile signals
- briefings

## SWFI Stack

launchd / managed service
→ official-source watchlist
→ governed research loop
→ narrow-expert AI router
→ review queue
→ profile signals / briefings / exports

## Product Boundary

Runtime Steward is an internal operating layer.
It supports:
- Profiles
- Key People
- Transactions
- Mandates and RFPs
- Asset Allocation
- Datafeeds and API access
- trust, provenance, and research operations

It is not a generic assistant and should not answer outside that lane.
