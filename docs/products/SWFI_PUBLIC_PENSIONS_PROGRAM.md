# SWFI Public Pensions Coverage Program

## Purpose

Turn the list of public pensions that actually allocate into a governed coverage and enrichment program.

## Why It Matters

This is not just a spreadsheet. It is:
- a coverage queue
- a contact enrichment queue
- an institutional prioritization list
- a mandate and RFP watch target set

## Minimum Record

Each pension record should carry:
- institution name
- city
- state
- assets
- key person status
- verified name
- verified email
- verified title
- SWFI update status
- export readiness
- provenance

## Program Stages

1. Normalize institution identity
2. Match existing SWFI profile
3. Attach key-person coverage
4. Mark trust status
5. Attach mandate/RFP watch eligibility
6. Promote to export-ready when minimum fields are present

## Product Uses

- SWFI coverage repair
- outbound target list building
- pension allocator benchmark set
- mandate timing watchlists
- SECI-adjacent public-institution benchmarking

## Audit Overlay

The program should carry a benchmark audit layer, not just a raw queue.

- overlay the target list against the current PI-300 pension cohort
- highlight top-asset institutions missing CIO / Executive Director coverage
- separate benchmark overlap from long-tail coverage
- use benchmark overlap to prioritize key-person enrichment and evidence collection
