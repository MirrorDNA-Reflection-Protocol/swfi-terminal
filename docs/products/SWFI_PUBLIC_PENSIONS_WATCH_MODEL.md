# SWFI Public Pensions Watch Model

## Position

Public pension coverage should be treated as an always-on source watch, not a static spreadsheet.

## What "Real-Time" Means Here

For pensions, real-time is a mixed model:

- same-day event coverage for board agendas, packets, videos, notices, RFPs, and leadership updates
- daily checks for official website changes and press releases
- weekly refresh checks for financial and actuarial report pages
- periodic refresh for structured public-plan datasets whose own publication cadence is quarterly or annual

SWFI should never claim real-time financials when the underlying source only updates periodically.

## Core Source Families

1. Official pension websites
2. Board and investment committee materials
3. RFP and procurement notices
4. Annual reports, CAFRs, and actuarial valuations
5. Key-person and leadership pages
6. Structured public pension datasets
7. Oversight and regulator pages

## Operating Loop

1. Check source freshness
2. Pull newly posted documents or notices
3. Extract entities, key people, transactions, mandates, consultants, and strategy signals
4. Score evidence and confidence
5. Route high-value changes into review
6. Publish reviewed changes to profiles, briefings, alerts, and exports

## Output Promise

Every pension watch output should clearly separate:

- event freshness
- financial-report freshness
- profile coverage
- key-person coverage
- mandate / RFP watch status
- evidence trail
