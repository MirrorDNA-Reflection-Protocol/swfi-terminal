# Active Mirror Shared Core

## Purpose

Active Mirror Shared Core is the reusable governed-intelligence substrate that can power multiple vertical products without sharing product identity or customer data.

It is the internal engine behind:
- SWFI
- Active Mirror internal operations
- future domain-specific intelligence systems

## Shared Services

### Runtime Steward
- observes source watches, queues, health, and readiness
- routes tasks through governed execution paths
- writes outcomes back into review, audit, and reporting surfaces

### TokenShield
- local-first routing
- cost and token budgets
- cache and compression
- privacy-aware cloud escalation

### Trust Layer
- provenance
- confidence
- evidence linking
- review gates
- audit trails

### Benchmark Factory
- methodology-driven scoring systems
- confidence bands
- mandate or cohort normalization
- report, dashboard, and API outputs

### Coverage OS
- source watchlists
- repair queues
- enrichment programs
- readiness scoring

### Report Engine
- board-ready briefs
- CSV exports
- JSON APIs
- recurring alert payloads

### Operator Console
- queue visibility
- source status
- review actions
- export history

## Reusable vs Product-Specific

Reusable core:
- routing
- risk policy
- budget policy
- cache
- trust envelopes
- review ledgers
- watch scheduling
- run history

Product-specific layer:
- ontology
- scoring methodology
- customer entitlements
- branding
- UI language
- export contracts
- source hierarchy rules where needed

## SWFI Reference Implementation

SWFI is the first serious vertical for the shared core.

In SWFI, the core powers:
- SWFI Runtime Steward
- governed SWFI AI
- official-source watches
- MSCI export readiness
- profile repair queue
- SECI benchmark planning
- public-pensions coverage program

## Non-Negotiable Rules

1. Shared core may be reused across products.
2. Product data planes stay isolated.
3. Product-specific prompts and scoring rules stay versioned and separate.
4. No client-facing product should expose internal orchestration language by default.
5. Trust, cost, privacy, and review are control-plane concerns, not optional features.
